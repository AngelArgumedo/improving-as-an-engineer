# Plan de Desarrollo: Junior → Top Global Engineer
**Objetivo:** Alcanzar nivel de élite en Backend/Systems Engineering
**Duración estimada:** 18-24 meses de estudio intensivo (3-4h diarias)

## Filosofía del Plan

Este plan está diseñado para construir **conocimiento profundo**, no solo superficial. Los mejores ingenieros del mundo (Google, Meta, top startups chinas/indias) tienen:

1. **Fundamentos sólidos como roca** - Entienden cómo funciona todo por debajo
2. **Habilidad de resolver problemas complejos** - No buscan en Google, derivan soluciones
3. **Mentalidad de sistemas** - Piensan en trade-offs, escalabilidad, failures
4. **Práctica deliberada constante** - 10,000+ horas de código intencional

## Estructura de Estudio (3-4h diarias)

### Distribución diaria recomendada:
- **1.5h:** Teoría profunda (CS fundamentals, lectura de papers, libros técnicos)
- **1.5h:** Práctica de algoritmos + implementación desde cero
- **1h:** Proyecto práctico aplicando conceptos (build from scratch)

### Revisión semanal:
- Sábados: Resolver problemas de system design
- Domingos: Leer código fuente de proyectos open source (Redis, PostgreSQL, etc)

## Mindset Crítico

**Lo que NO te hará mejor:**
- ❌ Tutorial hell (seguir tutoriales sin entender)
- ❌ Framework hopping (cambiar de tecnología constantemente)
- ❌ Aprendizaje superficial (solo saber usar APIs)
- ❌ Procrastinación con "aprender más tarde"

**Lo que SÍ te hará top global:**
- ✅ Implementar todo desde cero al menos una vez
- ✅ Leer código fuente de sistemas reales
- ✅ Entender el "por qué" no solo el "cómo"
- ✅ Resolver problemas difíciles diariamente
- ✅ Construir proyectos completos end-to-end
- ✅ Contribuir a open source de calidad

## Recursos Clave

**Libros obligatorios:**
1. "Designing Data-Intensive Applications" - Martin Kleppmann
2. "Computer Systems: A Programmer's Perspective" - Bryant & O'Hallaron
3. "The Algorithm Design Manual" - Steven Skiena
4. "Operating Systems: Three Easy Pieces" - Remzi Arpaci-Dusseau

**Plataformas de práctica:**
- LeetCode (Hard problems daily)
- Codeforces (para competencias de algoritmos)
- Project Euler (matemáticas + algoritmos)

**Código fuente para leer:**
- Redis (estructura de datos, event loop)
- PostgreSQL (query optimizer, storage)
- Linux kernel (sistemas operativos)
- NGINX (performance, concurrencia)

## Proyectos "Build From Scratch"

Durante estos 18-24 meses, construirás desde cero:

1. ✅ Tu propio HTTP server (entender networking, TCP/IP)
2. ✅ Database engine simple (B-trees, indexing, transactions)
3. ✅ Cache distribuido (consistencia, replicación)
4. ✅ Load balancer (algoritmos de distribución)
5. ✅ Message queue (concurrencia, persistencia)
6. ✅ Scheduler de tareas distribuido
7. ✅ Tu propio garbage collector simple
8. ✅ Compilador o intérprete simple

Cada proyecto te enseña cómo funcionan las cosas por dentro.
